# Phase Playbook

各フェーズの詳細な進行ガイド。

## Phase 0: キックオフ（5-10分）

**目的**: プロダクトの現状を把握し、ワークショップの方向性を決める。

**進行手順**:
1. プロダクト名・概要を確認
2. 現フェーズ（構想/開発/運用）を把握
3. 既存のビジョン・戦略があれば共有依頼
4. ワークショップのモードとゴールを合意

**判定基準**: プロダクトの輪郭が見えた状態で次へ進む。

## Phase 1: Vision — Mad Libs（15-25分）

**目的**: 明確で共感を呼ぶビジョンステートメントを共創する。

**進行手順**:
1. 対象者を特定（「誰の世界を変えたいか」）
2. 現状の不満を言語化（「何が受け入れられないか」）
3. 理想像を描く（「どんな世界を実現したいか」）
4. アプローチを定義（「どうやって実現するか」）
5. Mad Libsテンプレートに組み立て
6. 推敲・合意

**良いビジョンの判定基準**:
- 具体的な対象者が明記されている
- 現状への不満が感情レベルで伝わる
- 理想像が実現可能だが野心的
- アプローチがユニークかつ実行可能
- 読んだ人が「参加したい」と思える

**深掘りのタイミング**:
- 対象者が「みんな」「全員」のように曖昧な場合
- 問題が表面的（真因に至っていない）場合
- 理想像とアプローチの整合性が取れていない場合

## Phase 2: RDCL Strategy（20-30分）

**目的**: ビジョンを実現する戦略を4要素で構造化する。

**進行手順**:
1. Real Pain — Phase 1の問題を検証可能な形に具体化
2. Design — 痛みを解消する設計思想を定義
3. Capabilities — 必要な能力とギャップを把握
4. Logistics — 届け方と収益モデルを検討

**良い戦略の判定基準**:
- ビジョンの各要素と戦略が一対一で紐づく
- トレードオフが明示されている（何をやらないか）
- リスクが識別され対策が考えられている
- 検証可能な仮説の形になっている

**フェーズ間接続**: Phase 1のビジョンの「対象者」→Real Pain、「理想像」→Design、「アプローチ」→Capabilities/Logistics に対応。

## Phase 3: Vision vs Survival Prioritization（15-20分）

**目的**: 施策をビジョン貢献度とサバイバル必要度で分類し、優先順位を明確化。

**進行手順**:
1. 現在の主要施策を列挙（ユーザーから5-10個）
2. 各施策にVision(1-5)とSurvival(1-5)を付与
3. マトリクスに配置
4. Vision Debtの評価

**Vision Debt警告の基準**:
- Survivalが4-5かつVisionが1-2の施策が全体の50%超
- 直近3ヶ月の施策がすべてSurvival象限
- ビジョンに直結する施策が1つもない

**フェーズ間接続**: Phase 2のRDCLから導出された施策を入力とする。

## Phase 4: OHL Execution Plan（15-20分）

**目的**: 上位施策を仮説検証型の実行計画に分解する。

**進行手順**:
1. Phase 3で上位の施策を3-5個選択
2. 各施策のObjectiveをビジョンに紐づけて記述
3. 検証可能なHypothesisを設定
4. Learning Signalと計測方法を定義
5. TimeboxとOwnerを設定

**フェーズ間接続**: Phase 3の最優先象限の施策が入力。

## Phase 5: Culture Matrix（10-15分）

**目的**: チームの文化的慣行を可視化し改善点を特定する。

**進行手順**:
1. チームの主要慣行を列挙（5-8個）
2. 各慣行の充実度(1-5)と緊急度(1-5)を評価
3. 改善アクションを提案

**フェーズ間接続**: Phase 4の実行計画を支える文化基盤の確認。

## Phase 6: Product Disease Check（10-15分）

**目的**: 7つのProduct Diseaseのリスクを評価し予防策を立てる。

**進行手順**:
1. 各疾病の症状を提示し該当を確認
2. リスクレベル（L/M/H）を判定
3. M/Hの疾病に対して緩和策を議論

**フェーズ間接続**: 全フェーズの成果を俯瞰し、構造的なリスクを検出。

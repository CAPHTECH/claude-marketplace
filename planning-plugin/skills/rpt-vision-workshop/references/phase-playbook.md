# Phase Playbook

各フェーズの判定基準・接続関係の詳細ガイド。

## Phase 0: キックオフ

**目的**: プロダクトの現状を把握し、ワークショップの方向性を決める。

**判定基準**: プロダクトの輪郭が見えた状態で次へ進む。

## Phase 1: Vision — Mad Libs

**目的**: 明確で共感を呼ぶビジョンステートメントを共創する。

**良いビジョンの判定基準**:
- 具体的な対象者が明記されている
- 現状への不満が感情レベルで伝わる
- 理想像が実現可能だが野心的
- アプローチがユニークかつ実行可能
- 読んだ人が「参加したい」と思える

**深掘りのタイミング**:
- 対象者が「みんな」「全員」のように曖昧な場合
- 問題が表面的（真因に至っていない）場合
- 理想像とアプローチの整合性が取れていない場合

## Phase 2: RDCL Strategy

**目的**: ビジョンを実現する戦略を4要素（Real Pain / Design / Capabilities / Logistics）で構造化する。

**良い戦略の判定基準**:
- ビジョンの各要素と戦略が一対一で紐づく
- トレードオフが明示されている（何をやらないか）
- リスクが識別され対策が考えられている
- 検証可能な仮説の形になっている

**フェーズ間接続**: Phase 1のビジョンの「対象者」→Real Pain、「理想像」→Design、「アプローチ」→Capabilities/Logistics に対応。

## Phase 3: Vision vs Survival Prioritization

**目的**: 施策をビジョン貢献度とサバイバル必要度で分類し、優先順位を明確化。

**Vision Debt警告の基準**:
- Survivalが4-5かつVisionが1-2の施策が全体の50%超
- 直近3ヶ月の施策がすべてSurvival象限
- ビジョンに直結する施策が1つもない

**フェーズ間接続**: Phase 2のRDCLから導出された施策を入力とする。

## Phase 4: OHL Execution Plan

**目的**: 上位施策を仮説検証型の実行計画に分解する。

**フェーズ間接続**: Phase 3の最優先象限の施策が入力。

## Phase 5: Culture Matrix

**目的**: チームの文化的慣行を可視化し改善点を特定する。

**フェーズ間接続**: Phase 4の実行計画を支える文化基盤の確認。

## Phase 6: Product Disease Check

**目的**: 7つのProduct Diseaseのリスクを評価し予防策を立てる。

**フェーズ間接続**: 全フェーズの成果を俯瞰し、構造的なリスクを検出。

# Investigation

Use this route for debugging, root-cause analysis, feasibility checks, or uncertainty reduction.

## Plan Priorities

- state the leading hypotheses
- identify observations that would confirm or reject them
- define the minimum probes needed to reduce uncertainty
- defer fixes until evidence supports them

## Include

- current symptoms or questions
- hypotheses
- evidence to gather
- repro or observation steps
- likely fork points depending on what is observed
- exit criteria for ending the investigation

## Avoid

- jumping directly to implementation
- presenting one theory as established fact without evidence
- broad scans with no narrowing logic

## Good Shape

A strong investigation plan is falsifiable.
It says what would change the conclusion.

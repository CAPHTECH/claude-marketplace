# Review

Use this route for code review, design review, or risk review.

## Plan Priorities

- findings first
- severity and evidence before summaries
- focus on regressions, broken contracts, and missing validation
- separate confirmed issues from open questions

## Include

- review scope
- likely risk areas
- evidence sources
- expected finding format
- residual risk or testing gaps

## Avoid

- turning the review into an implementation plan
- burying the main findings under a long overview
- speculative criticism without evidence

## Good Shape

A strong review plan answers:

- what to inspect first
- what counts as a real finding
- how to report it clearly and concisely

# 質問テンプレート

各カテゴリから状況に応じて2-3問を選択。全質問する必要はない。

## 1. 技術的背景

### 必須
- **Q1.1**: 主に使っている言語・フレームワークは？
  - 例: "TypeScriptとReact"、"PythonとDjango"、"Go"
  - → 技術スタック設定、コード例の言語選択に反映

### 状況に応じて
- **Q1.2**: 今取り組んでいるプロジェクトの概要は？
  - 例: "ECサイトのバックエンド"、"社内ツール"
  - → プロジェクト固有CLAUDE.mdの判断材料

- **Q1.3**: 得意な領域と、あまり詳しくない領域は？
  - 例: "フロントエンドは得意、インフラは苦手"
  - → 説明の粒度調整（苦手領域は詳しく）

- **Q1.4**: 開発経験の目安は？（年数より「何を作ってきたか」）
  - 例: "5年、主にWebアプリ"、"1年、個人開発中心"
  - → 前提知識の調整

## 2. 開発スタイル

### 必須
- **Q2.1**: コードを書くとき、まず動くものを作る派？最初から構造を考える派？
  - 選択肢: 動くもの優先 / 設計優先 / ケースバイケース
  - → リファクタリング提案のタイミング調整

### 状況に応じて
- **Q2.2**: 新しい技術を試すのが好き？安定した方法を好む？
  - 選択肢: 新しい技術を積極的に / 安定重視 / バランス
  - → ライブラリ提案時の保守性考慮

- **Q2.3**: 一人で開発？チームで？
  - 選択肢: 個人 / チーム / 両方
  - → コードスタイル、ドキュメント重視度

- **Q2.4**: テストやドキュメントに対するスタンスは？
  - 選択肢: TDD / 後からテスト / 最小限 / 状況次第
  - → テスト生成の積極性

## 3. AI協働スタイル

### 必須
- **Q3.1**: AIにどこまで任せたい？
  - 選択肢:
    - コード生成まで全部
    - レビューと提案
    - 設計相談・壁打ち
    - ヒントだけ
  - → 出力の完成度調整

### 状況に応じて
- **Q3.2**: 完成したコードがほしい？アプローチの提示がほしい？
  - 選択肢: 完成コード / アプローチ提示 / 両方
  - → 出力形式の調整

- **Q3.3**: コードの説明はどの程度詳しくほしい？
  - 選択肢: 最小限 / ポイントのみ / 詳細
  - → 説明の冗長性調整

- **Q3.4**: 自分のコードに対して積極的に改善提案してほしい？聞かれたときだけ？
  - 選択肢: 積極的に / 聞かれたら / 重大な問題のみ
  - → レビューモードの設定

## 4. コミュニケーションの好み

### 必須
- **Q4.1**: 簡潔な回答がいい？詳しい説明がいい？
  - 選択肢: 簡潔 / 詳しく / 状況次第
  - → 応答の長さ調整

### 状況に応じて
- **Q4.2**: コードにコメントは多めに入れてほしい？最小限でいい？
  - 選択肢: 多め / 最小限 / 重要箇所のみ
  - → コメント生成ポリシー

- **Q4.3**: 言語設定は？（応答、コード、コメント）
  - 選択肢:
    - 全て日本語
    - 応答は日本語、コードは英語
    - 全て英語
  - → 言語設定

- **Q4.4**: 選択肢を複数提示してほしい？ベスト案を一つ？
  - 選択肢: 複数提示 / ベスト1つ / 重要な分岐点のみ複数
  - → 提案スタイル

## クイック質問セット（時間がない場合）

最低限これだけ聞けばCLAUDE.mdを生成可能:

1. 主な技術スタック？
2. AIに何を一番期待する？（コード生成/レビュー/相談）
3. 簡潔派？詳細派？
4. 言語設定は？（日本語/英語/混合）

## デフォルト値

質問をスキップした場合のデフォルト:

| 項目 | デフォルト値 |
|------|-------------|
| 言語 | 日本語 |
| 説明の詳しさ | 簡潔 |
| AI役割 | コード生成+レビュー |
| 改善提案 | 積極的 |
| コメント | 重要箇所のみ |
| 選択肢提示 | ベスト案1つ |

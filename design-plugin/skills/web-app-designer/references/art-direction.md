# アートディレクション・ビジュアルコンポジション リファレンス

ビジュアル構図、ブランド表現、コンテンツ戦略、モーション設計、画像ガイドラインの参照情報。
LP/マーケティングサイトとプロダクトUIの両方に適用される原則を体系化する。

---

## 共通原則

### ワンコンポジション

最初のビューポートは**単一の構図**として設計する。ダッシュボード風のカードモザイクにしない。

| ルール | 説明 |
|--------|------|
| ブランド優先 | ブランド名がナビゲーション以上の重要性を持つ |
| ブランドテスト | ナビゲーションを削除しても認識可能であるべき |
| 表現力のあるタイポグラフィ | デフォルトのシステムフォントスタックを避け、個性のあるフォントを選ぶ |
| 背景に深みを持たせる | 単色背景を避け、グラデーション・画像・パターンを使用 |
| セクション単一目的 | 各セクションは1つの目的と1つの見出しのみ |
| カード使用制限 | デフォルトはカードなし。相互作用が必要な場合のみカードを使用 |

### デザイン制約の事前定義

デザイン着手前に以下の制約を決定する:

| 制約 | 推奨上限 |
|------|----------|
| H1見出し | 1個 |
| セクション数 | 最大6個 |
| フォント種類 | 2種類まで |
| アクセント色 | 1色 |
| プライマリCTA | 1個（ファーストビューに配置） |

### コアデザイントークン

| トークン | 用途 |
|----------|------|
| background | ページ全体の背景 |
| surface | カード・パネル等の表面 |
| primary text | 主要テキスト |
| muted text | 補助テキスト |
| accent | 強調・CTA |

### タイポグラフィロール

| ロール | 用途 |
|--------|------|
| display | ヒーロー見出し、最大サイズ |
| headline | セクション見出し |
| body | 本文テキスト |
| caption | 注釈、補助情報 |

## ブランド主導面（LP・マーケティング）

### ビジュアル方針の事前定義

デザイン着手前に3つの方針を言語化する:

1. **Visual thesis**: ムード・素材・エネルギーを1文で説明
2. **Content plan**: ヒーロー → サポート → 詳細 → 最終CTAの構成
3. **Interaction thesis**: 2-3のモーションアイデア

### ナラティブ構造

ページを以下の5セクション構造で組み立てる:

| 順序 | セクション | 役割 |
|------|-----------|------|
| 1 | ヒーロー | アイデンティティと約束 |
| 2 | 支援画像 | 文脈・環境の提示 |
| 3 | 製品詳細 | オファーの説明 |
| 4 | ソーシャルプルーフ | 信用構築 |
| 5 | 最終CTA | コンバージョン |

### ヒーロー構成ルール

**必須要素**: ブランド名、約束（キャッチコピー）、CTA、ビジュアル（1つ）

**ビューポート予算**: スティッキーヘッダー + ヒーロー内容は初期ビューポート内に収める

**禁止事項**:
- ヒーローオーバーレイ（テキスト上の半透明レイヤー）
- 浮遊バッジ
- カード型ヒーロー
- 全幅不要な中央カラムヒーロー
- テキスト上の分割画面ヒーロー（一貫した側面配置は可）

**ヒーロー画像要件**:
- 製品・場所・雰囲気・文脈を表示する
- 画像を削除しても機能するなら、その画像は弱い
- テキスト上は強いコントラストと明確なタップターゲットを確保

### フルブリード

ランディングページは全幅の画像を基本とする。コンテナ幅に閉じ込めない。

## タスク主導面（プロダクトUI・ダッシュボード）

Linear風のミニマルアプローチを推奨する。

### 推奨パターン

| パターン | 説明 |
|----------|------|
| 落ち着いた表面階層 | 背景 → surface → elevated surface の控えめな階層 |
| 強いタイポグラフィ・間隔 | 装飾ではなくフォントサイズ・ウェイト・間隔で階層を表現 |
| 少ない色 | アクセント色は1色、セマンティックカラーは必要最小限 |
| 濃密だが読みやすい情報 | 情報密度を保ちつつ読みやすさを確保 |
| 最小限のクローム | 不要なボーダー・シャドウ・装飾を排除 |
| カードはインタラクション時のみ | 静的表示にカードを使わない |

### 非推奨パターン

| パターン | 理由 |
|----------|------|
| ダッシュボードカードモザイク | 情報の優先順位が不明確になる |
| すべての領域への太い枠線 | 視覚的ノイズが増える |
| ルーティンUIの装飾グラデーション | 注意を散漫にする |
| 複数の競合するアクセント色 | 視覚的階層が崩れる |
| 無駄なアイコン | 意味のないアイコンは認知負荷を増やす |

### ユーティリティコピー

プロダクトUIのコピーは以下の優先順位で書く:

1. **方向付け**: ユーザーが今どこにいるか
2. **ステータス**: 現在の状態は何か
3. **アクション**: 次に何ができるか

避けるべき表現:
- キャンペーン言語・比喩
- エグゼクティブサマリー風の文章
- 約束・気分・ブランドボイスの過度な強調

セクション見出し例: 「選択KPI」「計画ステータス」「検索メトリクス」

## 画像ガイドライン

### 基本原則

| 原則 | 説明 |
|------|------|
| 本物の画像を優先 | 現地写真 ＞ 抽象グラデーション・偽3D |
| 1枚以上の強い画像 | ブランド・会場・編集ページには「強く本物そっくりな画像」を必ず1枚以上 |
| テキスト用の落ち着いた色調 | 画像内にテキスト配置用の落ち着いた色調領域を選択 |
| 埋め込みテキスト回避 | 画像内の標識・ロゴ・タイプとの競合を避ける |
| 複数時点は複数画像 | コラージュではなく、個別の画像で表現 |

### 実質的ビジュアルアンカー

装飾グラデーションではなく、実際の製品・環境・人物の画像を使用する。ビジュアルアンカーは以下を満たすべき:

- 製品やサービスの文脈を伝える
- 削除するとページの意味が変わる（=不可欠である）
- テキストとの十分なコントラストを確保している

## コピーガイドライン

### LP・マーケティング向け

| ルール | 説明 |
|--------|------|
| 製品言語で書く | デザインコメンタリー（「美しい」「革新的」等の自己言及）を避ける |
| 見出しが意味を担う | 見出しだけをスキャンして内容が理解できること |
| 支援コピーは短い1文 | 説明文は1文で完結させる |
| セクション間の繰り返し削除 | 同じメッセージを異なる表現で繰り返さない |
| 各セクションに1責任 | 説明・証明・深化・変換のいずれか1つ |

### コピーの品質チェック

- すべての見出しだけを読んで、ページの主張が理解できるか？
- フィラーコピー（意味のない埋め合わせ文）がないか？
- ブランド名より大きな見出しがないか？（ブランドページの場合）

## モーション設計

### 意図的モーション原則

ページ全体で2-3の意図的なモーションに限定する。

### 必須3モーション（LP向け）

| モーション | 説明 |
|-----------|------|
| ヒーロー内進入シーケンス | ページロード時のヒーロー要素のアニメーション |
| スクロールリンク効果 | スティッキー要素・奥行き・パララックス効果 |
| ホバー・リビール・レイアウト遷移 | インタラクション時の応答アニメーション |

### 推奨実装パターン（Framer Motion）

| パターン | 用途 |
|----------|------|
| セクションリビール | スクロールで画面に入った時の表示アニメーション |
| 共有レイアウト遷移 | 要素間のスムーズな遷移 |
| スクロールリンク不透明度・変換・スケール | スクロール位置に連動した視覚効果 |
| スティッキーストーリーテリング | 固定要素とスクロールコンテンツの組み合わせ |
| ナラティブ進行カルーセル | ストーリーを進めるカルーセル |

### モーション品質チェック

| チェック項目 | 基準 |
|-------------|------|
| 目立つか | クイックレコーディングで確認して目立つこと |
| モバイル対応 | モバイルでもスムーズに動作すること |
| 速度 | 高速・控えめであること |
| 一貫性 | ページ全体で一貫したモーション言語であること |
| 必要性 | 装飾のみなら削除すること |

## アンチパターン（避けるべき失敗）

### 7つの失敗パターン

| # | パターン | 問題 |
|---|---------|------|
| 1 | 汎用SaaSカードグリッドが第一印象 | ブランドの個性が伝わらない |
| 2 | 美しい画像 + 弱いブランド | 何のサイトか分からない |
| 3 | 強い見出し + アクション不明確 | コンバージョンに繋がらない |
| 4 | テキスト背後の忙しい画像 | 可読性が損なわれる |
| 5 | セクション間で繰り返される気分説明 | 冗長で退屈 |
| 6 | ナラティブ目的のないカルーセル | 意味のないスワイプ |
| 7 | 積み重ねカード構成のアプリUI | カード濫用で情報階層が不明確 |

### 厳密な禁止事項

- デフォルトでカードを使わない
- ヒーロー内にカードを置かない
- セクションに複数の支配的アイデアを入れない
- ブランドページでブランドより大きな見出しを使わない
- フィラーコピーを入れない
- フォントを明確な理由なく2種類以上使わない

## 品質リトマステスト

デザイン完了時に以下を確認する。すべてYesであること:

| # | チェック項目 |
|---|-------------|
| 1 | ブランド/製品は最初の画面で誤認識不可能か？ |
| 2 | 強いビジュアルアンカーが1つ以上あるか？ |
| 3 | 見出しのスキャンのみでページの主張が理解できるか？ |
| 4 | 各セクションに1つの仕事しかないか？ |
| 5 | カードは本当に必要か？（なくても成立しないか確認） |
| 6 | モーションは視覚的階層や雰囲気を向上させているか？ |
| 7 | 装飾シャドウをすべて削除してもプレミアムに見えるか？ |

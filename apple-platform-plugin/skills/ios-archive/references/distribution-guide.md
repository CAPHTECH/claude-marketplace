# iOS配布ガイド

## 配布方法の比較

| 方法 | 対象 | 制限 | レビュー |
|------|------|------|---------|
| App Store | 一般ユーザー | なし | 必要 |
| TestFlight内部 | 開発チーム | 100人 | 不要 |
| TestFlight外部 | ベータテスター | 10,000人 | 初回のみ |
| Ad Hoc | 特定デバイス | 100台/年 | 不要 |
| Enterprise | 社内ユーザー | 社員のみ | 不要 |

## App Store提出チェックリスト

### 必須項目
- [ ] アプリ名（30文字以内）
- [ ] サブタイトル（30文字以内）
- [ ] 説明文（4,000文字以内）
- [ ] キーワード（100文字以内）
- [ ] スクリーンショット（各デバイスサイズ）
- [ ] アプリアイコン（1024x1024）
- [ ] プライバシーポリシーURL
- [ ] サポートURL

### スクリーンショットサイズ

| デバイス | サイズ |
|----------|--------|
| iPhone 6.7" | 1290 x 2796 |
| iPhone 6.5" | 1284 x 2778 |
| iPhone 5.5" | 1242 x 2208 |
| iPad Pro 12.9" | 2048 x 2732 |

## TestFlight配布

### 内部テスター
1. App Store Connectでユーザーを追加
2. ビルドをアップロード
3. テスターに自動通知

### 外部テスター
1. テスターグループを作成
2. ビルドをアップロード
3. Beta App Reviewに提出（初回のみ）
4. 承認後、テスター招待

### テスト情報の記載
```
テスト内容:
- 新機能: ダークモード対応
- バグ修正: ログイン画面のクラッシュ

既知の問題:
- 設定画面のプレースホルダーテキスト
```

## Ad Hoc配布

### UDID収集方法
1. デバイスをMacに接続
2. Finder/iTunesでデバイス情報を表示
3. シリアル番号をクリックしてUDID表示

### OTA配布手順
1. Ad Hocプロファイルにデバイス登録
2. IPAをエクスポート
3. manifest.plistを作成
4. HTTPSサーバーにアップロード
5. インストールリンクを配布

## Enterprise配布

### 要件
- Apple Developer Enterprise Program ($299/年)
- D-U-N-S番号
- 法人格

### 配布方法
- MDM (Mobile Device Management)
- 社内Webサイト（OTA）
- 社内アプリストア

### 注意点
- 社外への配布は規約違反
- 年次更新が必要
- 証明書失効時は全アプリ再署名が必要

# Mode Selection

## Light

Use when:

- local visual adjustment
- one component
- no major user journey change
- no irreversible user action
- no design-system change

## Standard

Use when:

- screen or flow design
- information architecture matters
- copy and layout interact
- user trust or confidence matters

## Deep

Use when:

- checkout, pricing, onboarding, auth, deletion, payment, AI autonomy, legal, healthcare, finance, or enterprise governance is involved
- multiple stakeholder relations conflict
- design-system primitives may change
- variants should be isolated and compared

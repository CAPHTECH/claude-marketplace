# Retraction Playbook

## Common retractions

### User segment changed

Impact:

- copy tone
- proof type
- information density
- terminology
- navigation priority

### Risk level changed

Impact:

- confirmation steps
- preview panels
- explanation before CTA
- error prevention

### Business goal changed

Impact:

- CTA hierarchy
- section order
- conversion path
- analytics events

### Design-system constraint changed

Impact:

- component selection
- token usage
- responsive behavior
- accessibility baseline

## Retraction decision categories

- keep: decision remains valid under new premise.
- revise: decision intent remains valid, implementation or copy must change.
- retract: decision is invalid and should be removed.
- unresolved: dependency cannot be evaluated from available evidence.

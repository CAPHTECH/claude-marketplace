# 手法カタログ（Method Catalog）

7カテゴリ・20手法の詳細リファレンス。

---

## 1. 学術・研究分野

### 1.1 文献調査（Literature Review）

- **概要**: 特定のテーマに関する既存の研究論文・書籍・報告書を体系的に収集・整理・分析する手法
- **使いどころ**: 研究の初期段階で先行研究を把握するとき、既知の知見を網羅的に把握したいとき
- **手順**:
  1. リサーチクエスチョンの設定
  2. 検索キーワード・データベースの決定（Google Scholar、PubMed、CiNii等）
  3. 文献の収集とスクリーニング
  4. 重要文献の精読とメモ作成
  5. テーマ別の整理と統合

### 1.2 一次情報と二次情報の使い分け

- **概要**: 一次情報（オリジナルデータ、インタビュー、実験結果）と二次情報（既存の論文、統計、報告書）を目的に応じて使い分ける考え方
- **使いどころ**: 調査設計時に「自分で取りに行くべきか、既存データで十分か」を判断するとき
- **手順**:
  1. 調査目的と必要な情報の明確化
  2. 既存の二次情報（統計、論文、レポート）を先に調査
  3. 二次情報で不足する部分を特定
  4. 一次情報（インタビュー、アンケート、実験等）で補完
  5. 両者を統合して結論を導出

### 1.3 系統的レビュー（Systematic Review）

- **概要**: あらかじめ定めた適格基準に基づき、特定の研究課題に関するすべての関連エビデンスを体系的に収集・評価する文献研究
- **使いどころ**: 特定の問いに対して「現時点での最良のエビデンスは何か」を確定させたいとき
- **手順**:
  1. リサーチクエスチョンの設定（PICO/PECOフレームワーク等）
  2. 検索戦略の策定と文献検索（複数データベースを使用）
  3. 適格基準に基づく文献のスクリーニング
  4. 研究の質の評価（バイアスリスク評価）
  5. エビデンスの要約と統合
  6. 結果の解釈と報告（PRISMA声明に準拠）

### 1.4 メタ分析（Meta-Analysis）

- **概要**: 系統的レビューで収集した複数の研究結果を統計的に統合し、効果量の推定や異質性の評価を行う手法
- **使いどころ**: 同一テーマの複数の研究結果を定量的に統合して全体的な結論を出したいとき
- **手順**:
  1. 系統的レビューに基づく研究の選定
  2. 各研究からのデータ抽出
  3. 効果量の算出（オッズ比、リスク比、平均差等）
  4. 異質性の評価（I²統計量等）
  5. 統合効果量の算出（固定効果モデル/変量効果モデル）
  6. 出版バイアスの評価（ファンネルプロット等）

## 2. ビジネス・インテリジェンス

### 2.1 デスクリサーチ（机上調査）

- **概要**: 既存の公開情報を収集・分析する調査手法。官公庁統計、業界団体の報告書、企業のIR資料、ニュース記事等を活用
- **使いどころ**: 市場規模の把握、業界動向の概観、競合の基本情報収集など、調査の初期段階
- **手順**:
  1. 調査目的・仮説の設定
  2. 情報源の特定（政府統計、業界レポート、IR資料、ニュース等）
  3. データの収集
  4. データの整理・分析
  5. 仮説の検証・修正
  6. フィールドリサーチが必要な領域の特定

### 2.2 フィールドリサーチ（現地調査）

- **概要**: 現地訪問、インタビュー、アンケート、観察等により一次情報を直接取得する手法
- **使いどころ**: ユーザーの行動・心理・動機を深く理解したいとき、デスクリサーチでは得られない情報を取得したいとき
- **手順**:
  1. 調査設計（対象者、手法、サンプルサイズの決定）
  2. 調査票/インタビューガイドの作成
  3. 現地でのデータ収集（インタビュー、観察、アンケート等）
  4. データの整理・分析
  5. インサイトの抽出と報告

### 2.3 競合調査（ベンチマーク調査）

- **概要**: 競合企業の戦略、製品、価格、マーケティング、組織体制などを多角的に調査・分析する手法
- **使いどころ**: 新規事業立ち上げ、新製品開発、販促強化の際に差別化戦略を立案するとき
- **手順**:
  1. 競合の特定と優先順位付け
  2. 調査項目の設定（製品、価格、チャネル、プロモーション、技術力、財務状況等）
  3. 公開情報の収集（Webサイト、IR、特許、求人情報等）
  4. 比較分析（自社 vs 競合のマトリクス作成）
  5. 差別化ポイントと参入障壁の特定
  6. 戦略提言

### 2.4 市場調査（マーケティングリサーチ）

- **概要**: 市場の規模、動向、顧客ニーズ、競争環境を体系的に調査する手法。デスクリサーチとフィールドリサーチの両方を含む
- **使いどころ**: 新規市場参入、製品開発、マーケティング戦略策定の意思決定材料として
- **手順**:
  1. 調査目的の明確化
  2. 二次データによるデスクリサーチ（市場規模、トレンド）
  3. 仮説の設定
  4. 一次データの収集（アンケート、インタビュー、フォーカスグループ）
  5. データ分析（統計分析、セグメンテーション等）
  6. 報告・意思決定への反映

## 3. ジャーナリズム・ファクトチェック

### 3.1 取材手法（ジャーナリスティック・リサーチ）

- **概要**: ジャーナリストが情報を収集・検証するための体系的手法。直接取材、資料収集、データ分析を組み合わせる
- **使いどころ**: ニュース報道、調査報道、ドキュメンタリー制作、取材活動
- **手順**:
  1. テーマ・仮説の設定
  2. 背景情報の収集（デスクリサーチ）
  3. 情報源の特定とアプローチ
  4. 直接取材（インタビュー、現地調査）
  5. 情報の裏取り（複数ソースでの確認）
  6. 事実と推測の明確な区別
  7. 原稿作成と編集チェック

### 3.2 ファクトチェック

- **概要**: 流通する言説が事実に基づいているかを検証し、正確性を判定する手法
- **使いどころ**: デマ・誤情報の検証、報道の信頼性確認、研究データの妥当性チェック
- **手順**:
  1. 検証対象の特定（拡散している主張・言説の選定）
  2. ソース確認（元の情報源の特定）
  3. 疑わしい兆候の発見（論理的矛盾、数値の不整合等）
  4. 他メディアや公的機関の情報での確認
  5. 一次情報（元データ、統計原典等）の確認
  6. 判定と結果の公開（根拠とともに）

### 3.3 情報源の信頼性評価

- **概要**: 情報の出所（ソース）が信頼に足るかを体系的に評価する考え方
- **使いどころ**: 調査・報道の情報源選定、引用する資料の信頼度判断
- **手順**:
  1. 一次情報かどうかの確認
  2. 発信主体の特定（著者、組織、編集体制）
  3. 更新日時・版・改定履歴の確認
  4. 利害関係・動機の分析（商業的、政治的バイアスの有無）
  5. 他の信頼できる情報源との照合
  6. 専門性・権威性の評価

## 4. 問題解決フレームワーク

### 4.1 5W1H

- **概要**: Who / What / When / Where / Why / How の6つの問いで情報を網羅的に整理するフレームワーク
- **使いどころ**: 問題の全体像把握、企画書作成、要件定義、コミュニケーションの構造化
- **手順**:
  1. テーマ・問題を設定
  2. 各Wの問いに答える形で情報を整理
  3. 不足している情報を特定
  4. 追加調査で情報を補完
  5. 全体像を統合して結論を導出

### 4.2 なぜなぜ分析（5 Whys / RCA）

- **概要**: 問題事象に対して「なぜ？」を繰り返す（通常3〜5回）ことで根本原因（Root Cause）に到達する手法。トヨタ自動車が開発
- **使いどころ**: バグの根本原因調査、障害分析、品質管理、再発防止策の策定
- **手順**:
  1. 問題事象を具体的に記述する
  2. 「なぜその問題が起きたのか？」→ 直接原因を記述
  3. 直接原因に対して「なぜ？」→ より深い原因を記述
  4. 3〜5回繰り返し、根本原因に到達
  5. 根本原因に対する再発防止策を策定
  6. 防止策の実行と効果検証

### 4.3 デザイン思考（Design Thinking）

- **概要**: スタンフォード大学d.schoolが提唱する人間中心の問題解決アプローチ。共感→定義→発想→試作→検証の5ステップを反復する
- **使いどころ**: ユーザー体験の改善、新製品/サービスの開発、イノベーション創出
- **手順**:
  1. **共感（Empathize）**: ユーザーインタビュー、観察を通じて深く理解
  2. **定義（Define）**: 本質的な課題を「問題定義文」として明確化
  3. **発想（Ideate）**: ブレインストーミング等で多数の解決策を創出
  4. **試作（Prototype）**: アイデアを素早く形にして試作品を作成
  5. **検証（Test）**: ユーザーに試作品を使ってもらいフィードバックを収集
  6. 必要に応じてステップ1に戻り反復

### 4.4 仮説思考（Hypothesis-Driven Thinking）

- **概要**: 限られた情報から「仮の結論（仮説）」を設定し、実行・検証・修正を繰り返す思考法。コンサルティング業界で広く活用
- **使いどころ**: 時間制約のあるビジネス課題、不確実性の高い意思決定、網羅的調査の前に方向性を絞りたいとき
- **手順**:
  1. 状況の観察・分析（限られた情報の把握）
  2. 仮説の設定（「こうではないか？」という仮の結論）
  3. 仮説を検証するために必要なデータ・事実の特定
  4. データ収集と分析による仮説の検証
  5. 仮説の修正または確定
  6. 必要に応じてサイクルを繰り返す

## 5. デジタル調査

### 5.1 OSINT（Open Source Intelligence）

- **概要**: 公開されている情報源（Webサイト、SNS、ニュース、公的記録、学術論文等）から情報を収集・分析し、実用的なインテリジェンスに変換する手法
- **使いどころ**: セキュリティ調査、競合インテリジェンス、デューデリジェンス、法的調査
- **手順**:
  1. 調査目的とインテリジェンス要件の定義
  2. 情報源の特定（Web、SNS、公的データベース、フォーラム等）
  3. データの収集（手動検索 + ツール活用）
  4. データの分析・相関付け
  5. インテリジェンスレポートの作成
  6. 情報の信頼度評価と検証
- **主要ツール**: OSINT Framework、SpiderFoot、Maltego、Shodan、Google Dorks

### 5.2 Webアーカイブ活用

- **概要**: Internet Archive（Wayback Machine）等を利用して、過去のWebページの状態を調査する手法
- **使いどころ**: 過去のWebサイト内容の確認、削除されたコンテンツの復元、Webの時系列変化の分析
- **手順**:
  1. 調査対象URLの特定
  2. Wayback Machine（web.archive.org）でのスナップショット検索
  3. 時系列でのコンテンツ変化の比較分析
  4. 削除・変更された情報の記録
  5. 他のソースとの照合による検証

### 5.3 Google高度検索（Advanced Search Operators）

- **概要**: Google検索の高度な演算子（site:、filetype:、intitle:、inurl:等）を活用して精度の高い情報検索を行う手法
- **使いどころ**: 特定ドメイン内の情報検索、特定ファイル形式の検索、OSINT調査
- **手順**:
  1. 検索目的の明確化
  2. 適切な検索演算子の選択と組み合わせ
  3. 検索結果のスクリーニング
  4. 有用な情報の抽出と記録
  5. 追加の検索クエリでの補完
- **主な演算子**:
  - `site:example.com` — 特定ドメイン内を検索
  - `filetype:pdf` — 特定ファイル形式を検索
  - `intitle:"キーワード"` — タイトルにキーワードを含むページ
  - `inurl:keyword` — URLにキーワードを含むページ
  - `"完全一致フレーズ"` — 完全一致検索
  - `-除外ワード` — 特定ワードを除外
  - `before:YYYY-MM-DD` / `after:YYYY-MM-DD` — 日付指定

## 6. 情報評価

### 6.1 CRAAP Test

- **概要**: California State University, Chicoが開発した情報源の信頼性評価フレームワーク。5基準で評価
- **使いどころ**: 調査で得た情報源の信頼性判断、引用資料の選定
- **手順**:
  1. **Currency（通貨性/新しさ）**: いつ公開・更新されたか？情報は最新か？
  2. **Relevance（関連性）**: テーマに直接関係するか？対象読者は？
  3. **Authority（権威性）**: 著者は誰か？資格・所属は？出版元は信頼できるか？
  4. **Accuracy（正確性）**: 根拠やエビデンスはあるか？査読済みか？
  5. **Purpose（目的）**: 情報提供・教育・販売・娯楽・説得のいずれか？バイアスの有無は？

詳細な評価チェックリストは `evaluation-guide.md` を参照。

### 6.2 バイアスの排除

- **概要**: 調査・分析過程で生じる認知バイアス（確証バイアス、選択バイアス、生存者バイアス等）を意識的に排除・低減する手法
- **使いどころ**: あらゆる調査・分析活動において常に意識すべき横断的テーマ
- **手順**:
  1. 自身の前提・仮定を明示する（暗黙の仮定を可視化）
  2. 反証を積極的に探す（自説を否定する情報も収集）
  3. 多様な情報源を使う（同じタイプのソースに偏らない）
  4. 複数人でのクロスチェック（ピアレビュー）
  5. プレモーテム（「この結論が間違っているとしたら？」と問う）
  6. メタ認知の実践（自分の思考プロセス自体を振り返る）

### 6.3 SIFT法（情報の迅速評価）

- **概要**: Stop / Investigate / Find better coverage / Trace claims の4ステップで情報をすばやく評価する手法。Mike Caulfield提唱
- **使いどころ**: SNSやWebで見かけた情報の即時評価、情報の真偽の迅速判定
- **手順**:
  1. **Stop（止まる）**: 反射的に拡散しない。まず立ち止まる
  2. **Investigate the source（情報源を調べる）**: 発信者の素性と信頼性を確認
  3. **Find better coverage（より良い情報を探す）**: 同じ事実を扱う他の信頼できる情報源を確認
  4. **Trace claims（主張を遡る）**: 元の情報源・一次情報を辿る

詳細は `evaluation-guide.md` を参照。

## 7. 整理・統合手法

### 7.1 KJ法（親和図法 / Affinity Diagram）

- **概要**: 文化人類学者・川喜田二郎が考案した情報整理手法。断片的なデータをカードに書き出し、類似性に基づいてグループ化・構造化する
- **使いどころ**: ブレインストーミング結果の整理、定性調査データの分析、ユーザーインタビューの整理
- **手順**:
  1. テーマの設定
  2. 個別データをカード（付箋）1枚1項目で書き出す
  3. カードを広げて全体を俯瞰
  4. 類似するカードをグループ化（直感的に）
  5. 各グループに見出しをつける
  6. グループ間の関係性を図解
  7. 文章化（図解をもとに構造的な説明を作成）

### 7.2 マインドマップ

- **概要**: トニー・ブザンが提唱した視覚的思考ツール。中心テーマから放射状にブランチを伸ばし、関連する情報を階層的に展開する
- **使いどころ**: アイデア発散、調査結果の視覚化、学習内容の整理、プレゼンの構成設計
- **手順**:
  1. 中心テーマを紙（またはツール）の中央に配置
  2. メインブランチ（大カテゴリ）を放射状に展開
  3. サブブランチ（詳細項目）を各メインブランチから展開
  4. キーワード、色、イメージを活用して情報を記述
  5. ブランチ間の関連をクロスリンクで表現
  6. 全体を俯瞰して構造を調整

### 7.3 エビデンスピラミッド（Hierarchy of Evidence）

- **概要**: 研究デザインの信頼性をピラミッド型で可視化した概念。下位から上位へ信頼度が上がる
- **使いどころ**: 引用するエビデンスの強度判断、研究結果の信頼度比較
- **手順（活用方法）**:
  1. 手元のエビデンスの研究デザインを分類
  2. ピラミッド上の位置を確認
  3. 上位のエビデンスが利用可能か検索
  4. 同一レベルのエビデンス間では研究の質を比較
  5. エビデンスの強さを踏まえて結論の確信度を調整

信頼度の階層（上位ほど信頼度が高い）:
1. 系統的レビュー / メタ分析
2. ランダム化比較試験（RCT）
3. コホート研究
4. 症例対照研究
5. 症例報告 / 症例集積
6. 専門家の意見 / 社説

### 7.4 MECE（Mutually Exclusive, Collectively Exhaustive）

- **概要**: 「漏れなく、ダブりなく」情報を分類・整理するための論理的思考の基本原則
- **使いどころ**: 調査結果の分類、問題の構造化、戦略の選択肢洗い出し
- **手順**:
  1. 分類対象（全体集合）を定義
  2. 分類の切り口（軸）を設定
  3. 各カテゴリが相互に排他的（重複なし）であることを確認
  4. 全カテゴリの合計が全体を網羅している（漏れなし）ことを確認
  5. 必要に応じて切り口を変更・調整

### 7.5 ロジックツリー

- **概要**: MECE原則に基づき、テーマを上位概念から下位概念へツリー状に分解して可視化する手法
- **使いどころ**: 問題の構造化、原因分析、解決策の網羅的洗い出し
- **手順**:
  1. 最上位の問い（テーマ）を設定
  2. 第一階層をMECEに分解
  3. 各要素をさらに下位階層へ分解
  4. 3〜4階層程度まで掘り下げ
  5. 各末端要素について分析・評価
  6. 全体を俯瞰してパターンや優先順位を特定
- **3つの型**:
  - **Why型（原因探索）**: 「なぜこの問題が起きているのか？」
  - **What型（要素分解）**: 「この全体は何で構成されているか？」
  - **How型（解決策探索）**: 「どうすれば解決できるか？」

# Observation methods catalog

## 現物観測（Prototype / PoC）
- 目的: 体験・成立性・性能の早期確認
- 証拠: 動くデモ、動画、計測値、クラッシュログ
- 向く不確実性: UX/要件解釈/実現性/性能

## 証拠観測（Logs / Metrics / Tests）
- 目的: 既存事実の把握、主張の裏付け
- 証拠: ログ、ダッシュボード、テスト結果、再現手順
- 向く不確実性: バグ原因、性能劣化、品質、運用

## 知識観測（Docs / Stakeholder Q&A）
- 目的: 外部制約や仕様の確認
- 証拠: 公式ドキュメント引用、関係者回答、審査ガイドライン
- 向く不確実性: 法務/審査/依存API/契約

## 最小観測の原則
- 観測結果が「次の意思決定」を変えるサイズに切る
- タイムボックスを先に決める（30分/2h/半日）
- 判定規則を先に書く（結果Aなら採用、Bなら撤回）

# 情報評価ガイド（Evaluation Guide）

情報の信頼性を評価するための詳細リファレンス。

---

## 1. CRAAP Test

California State University, Chicoの図書館員Sarah Blakeslee（2004年）が開発した情報源の信頼性評価フレームワーク。

### Currency（通貨性/新しさ）

情報の鮮度を評価する。

| チェック項目 | 確認すること |
|-------------|-------------|
| 公開日 | いつ公開された情報か？ |
| 更新日 | 最終更新はいつか？ |
| リンクの有効性 | 参照リンクは生きているか？ |
| 分野との適合 | その分野で許容される古さか？（法律: 最新が必須、歴史: 古くてもOK） |

**判定基準**:
- 高: 過去1〜2年以内に公開/更新、分野に対して十分に新しい
- 中: 3〜5年前だが、大きな変化がない分野
- 低: 5年以上前で、分野が急速に変化している

### Relevance（関連性）

情報がテーマに適合しているかを評価する。

| チェック項目 | 確認すること |
|-------------|-------------|
| テーマとの関連 | 調査テーマに直接関係するか？ |
| 対象読者 | 誰向けに書かれた情報か？（専門家 / 一般向け / 学生向け） |
| 情報の深さ | 必要な深さの情報が得られるか？ |
| 代替情報源 | より適切な情報源がないか？ |

### Authority（権威性）

情報の発信者・出版元の信頼性を評価する。

| チェック項目 | 確認すること |
|-------------|-------------|
| 著者の特定 | 著者は明記されているか？ |
| 資格・専門性 | 著者はそのテーマの専門家か？ |
| 所属組織 | 信頼できる組織に所属しているか？ |
| 連絡先 | 著者や出版元に問い合わせ可能か？ |
| ドメイン | .edu / .gov / .org / .com のどれか？ |

**ドメイン別の一般的信頼度**:
- `.gov` / `.go.jp`: 政府機関（高）
- `.edu` / `.ac.jp`: 教育機関（高）
- `.org`: 非営利団体（中〜高、団体による）
- `.com`: 商業サイト（要確認）

### Accuracy（正確性）

情報の正確さと根拠を評価する。

| チェック項目 | 確認すること |
|-------------|-------------|
| 根拠の有無 | 主張にエビデンスや参考文献が示されているか？ |
| 査読の有無 | 査読プロセスを経ているか？ |
| 他情報との整合 | 他の信頼できる情報源と矛盾しないか？ |
| 事実と意見の区別 | 事実と意見が明確に区別されているか？ |
| 誤字・論理の飛躍 | 文章の品質に問題はないか？ |

### Purpose（目的）

情報が発信された目的とバイアスを評価する。

| チェック項目 | 確認すること |
|-------------|-------------|
| 発信目的 | 情報提供 / 教育 / 販売 / 娯楽 / 説得のどれか？ |
| バイアスの有無 | 特定の立場に偏っていないか？ |
| 利害関係 | 著者や出版元に経済的・政治的利害はないか？ |
| 客観性 | 事実を客観的に提示しているか？ |

### CRAAP Test スコアリングシート

各基準を1〜5点で評価し、合計点で総合判定する:

| 基準 | 1（低） | 3（中） | 5（高） |
|------|---------|---------|---------|
| Currency | 10年以上前、更新なし | 3〜5年前 | 1年以内、定期更新 |
| Relevance | テーマと間接的にしか関連しない | 部分的に関連 | テーマに直接関連 |
| Authority | 著者不明、個人ブログ | 業界関係者 | 専門家、学術機関 |
| Accuracy | 根拠なし、矛盾あり | 一部根拠あり | 査読済み、エビデンス豊富 |
| Purpose | 広告・販促目的 | 教育目的だがバイアスあり | 客観的な情報提供 |

**合計点の目安**:
- 20〜25点: 信頼度 高 — 主要根拠として使用可能
- 13〜19点: 信頼度 中 — 補助的な根拠として使用、裏取り推奨
- 5〜12点: 信頼度 低 — 参考程度に留める、他の情報源で裏取り必須

## 2. SIFT法

Mike Caulfieldが提唱する情報の迅速評価手法。CRAAPテストよりも素早く適用でき、日常的な情報評価に適する。

### Stop（止まる）

最初にすべきことは「立ち止まること」。

- 情報を見た瞬間の感情的反応に気づく
- 「怒り」「驚き」「不安」を感じたら特に注意
- 反射的に拡散・共有しない
- 30秒でいいので、次のステップに進む時間を取る

### Investigate the source（情報源を調べる）

記事やポストの内容を読む前に、発信者を調べる。

- 発信者は誰か？（個人 / 組織 / メディア）
- その分野の専門家か？
- 過去の発信に問題はないか？
- Wikipediaで発信元の組織を確認（背景、資金源、評判）

### Find better coverage（より良い情報を探す）

同じ事実を扱う他の情報源を探す。

- 複数の独立した情報源が同じことを報告しているか？
- 信頼できるメディアや専門機関が取り上げているか？
- 元の情報源だけが報じている場合は注意
- Google Newsで同じトピックを検索

### Trace claims（主張を遡る）

情報の元ネタ（一次情報）を辿る。

- 引用されている研究や統計の原典にあたる
- 「研究によると」→ どの研究？誰の？いつの？
- 写真や動画の元ネタを確認（逆画像検索等）
- 文脈が切り取られていないか確認

### SIFT法の適用例

```
見かけた情報: 「新しい研究で◯◯が判明」というSNS投稿

1. Stop: 立ち止まる。すぐにリツイートしない
2. Investigate: 投稿者は誰？科学者？ニュースサイト？個人？
3. Find: Googleで同じ研究結果を検索。Nature、Science等の科学誌で報じられているか？
4. Trace: 元の論文を探す。査読済みか？サンプルサイズは？結論は投稿の主張と一致するか？
```

## 3. バイアスの排除手法

### 確証バイアス（Confirmation Bias）

**定義**: 自分が信じていることを支持する情報ばかりを集め、反する情報を無視する傾向。

**排除手法**:
- **反証探索**: 意図的に自説に反する情報を探す。「もし自分の結論が間違っているとしたら、どんなデータが見つかるか？」と問う
- **プレモーテム**: チームで「この結論が大失敗したとしたら、原因は何か？」と想定する
- **デビルズ・アドボケイト**: チーム内で意図的に反対意見を述べる役割を設ける
- **情報源の多様化**: 意識的に異なる立場の情報源を含める

### 選択バイアス（Selection Bias）

**定義**: 情報収集の対象やサンプルが偏っていることで、結論が歪む傾向。

**排除手法**:
- **情報源チェックリスト**: 使用した情報源の種類を記録し、偏りがないか確認する
- **網羅的検索**: 特定のデータベースだけでなく、複数の異なるソースを使う
- **否定的結果の考慮**: 「見つからなかった」情報も重要。出版バイアス（成功事例のみ公開される傾向）に注意
- **代表性の確認**: サンプルが母集団を適切に代表しているか検討する

### 生存者バイアス（Survivorship Bias）

**定義**: 成功した事例（生存者）だけを見て結論を出し、失敗して消えた事例を無視する傾向。

**排除手法**:
- **失敗事例の探索**: 成功事例と同じ条件で始まったが失敗した事例を探す
- **ベースレートの確認**: 成功率の母数（全体の試行数）を確認する
- **消えた事例の追跡**: 「以前はあったが今はない」事例に注目する
- **反実仮想**: 「もし全員が同じことをしたら、全員が同じ結果を得るか？」と問う

### バイアス対策チェックリスト

調査の各フェーズで以下を確認する:

| フェーズ | チェック項目 |
|---------|-------------|
| 調査設計 | 結論を先に決めていないか？ |
| 情報収集 | 都合の良い情報ばかり集めていないか？ |
| 情報選択 | 特定の情報を無視していないか？ |
| 分析 | データの解釈に自分の期待が影響していないか？ |
| 結論 | 反証に耐えうる結論か？ |

## 4. エビデンスピラミッド（Hierarchy of Evidence）

研究デザインの信頼性を階層化した概念。医療分野で確立されたが、他分野にも応用可能。

### 信頼度の階層

```
         ▲
        /  \        系統的レビュー / メタ分析
       /    \       （最も信頼度が高い）
      /------\
     /        \     ランダム化比較試験（RCT）
    /----------\
   /            \   コホート研究
  /--------------\
 /                \ 症例対照研究
/------------------\
      症例報告 / 症例集積
========================
   専門家の意見 / 社説
  （最も信頼度が低い）
```

### 各レベルの説明

| レベル | 研究デザイン | 信頼度 | 特徴 |
|--------|------------|--------|------|
| 1 | 系統的レビュー / メタ分析 | 最高 | 複数の研究結果を体系的に統合。バイアス最小化 |
| 2 | ランダム化比較試験（RCT） | 高 | 無作為割り付けで交絡因子を排除 |
| 3 | コホート研究 | 中〜高 | 前向きに追跡。因果関係の推定が可能 |
| 4 | 症例対照研究 | 中 | 結果から原因を遡る。稀な疾患に有用 |
| 5 | 症例報告 / 症例集積 | 低〜中 | 個別事例の記録。仮説生成に有用 |
| 6 | 専門家の意見 / 社説 | 低 | エビデンスに基づかない場合がある |

### ビジネス分野への応用

| 信頼度 | ビジネスでの対応 |
|--------|----------------|
| 最高 | 複数の独立した調査・分析の統合（業界横断レポート） |
| 高 | A/Bテスト、対照実験 |
| 中〜高 | 長期追跡調査、パネルデータ分析 |
| 中 | 事後分析、ベンチマーク比較 |
| 低〜中 | 個別のケーススタディ、成功事例 |
| 低 | コンサルタントの意見、ブログ記事、個人の経験 |

### 注意点

- 上位の研究デザインでも実施の質が低ければ信頼度は下がる
- 文脈によっては下位の研究が上位より適切な場合もある
- エビデンスの「レベル」と「質」は別の概念

## 5. 情報源ごとの信頼度目安表

| 情報源 | 信頼度 | 注意点 |
|--------|--------|--------|
| 査読付き学術論文 | 高 | 査読の質、ジャーナルのインパクトファクターにより差がある |
| 政府統計・公的機関の報告書 | 高 | 調査方法と定義を確認する。国によって基準が異なる |
| 国際機関の報告書（WHO、World Bank等） | 高 | データの出所と集計方法を確認 |
| 業界団体のレポート | 中〜高 | 業界の利害を代弁している可能性に注意 |
| 大手メディアの報道 | 中〜高 | 編集方針によるバイアスあり。複数メディアで確認 |
| 企業のIR資料・プレスリリース | 中 | 自社に有利な情報に偏る傾向。数値データは比較的信頼可能 |
| 書籍（専門書） | 中〜高 | 著者の専門性と出版年に注意 |
| 専門家インタビュー | 中 | 個人の意見であり、エビデンスではない |
| ニュースサイト・Webメディア | 低〜中 | 編集体制、ファクトチェック体制による |
| Wikipedia | 低〜中 | 出発点としては有用。引用元を辿って一次情報を確認する |
| 個人ブログ・SNS投稿 | 低 | 裏取り必須。発信者の専門性を確認 |
| AI生成コンテンツ | 低 | ハルシネーションの可能性。必ず一次情報で裏取り |
| 匿名掲示板・口コミ | 最低 | 情報の出所が不明。参考程度に留める |
